# Architecture

FormSight / 见相 treats a person as a simulated world.

## Pipeline

```text
1. Seed Intake
2. Signal Extraction
3. Autobiographical Graph Construction
4. Event Sedimentation Layer
5. Inner Agent Initialization
6. Personal World State Construction
7. Conflict Rule Matching
8. Life Environment Setup
9. Parallel Trajectory Simulation
10. Report Synthesis
```

## 1. Seed Intake

The seed is not a personality questionnaire.

It can contain:

- autobiographical fragments
- current dilemma
- repeated patterns
- recent events
- desired futures
- feared futures
- relationship observations
- career history
- diary excerpts
- chat excerpts

## 2. Signal Extraction

The engine extracts high-signal patterns:

- repeated verbs
- fear language
- desire language
- relational behavior
- decision behavior
- avoidance behavior
- identity claims

## 3. Autobiographical Graph

The graph models:

- self-beliefs
- important events
- repeated patterns
- people and institutions
- desires and fears
- constraints and resources

## 4. Event Sedimentation Layer

The engine should not treat a person's past as background story only.

Life events are converted into age-sensitive deposits:

- early family pressure
- school and peer evaluation
- romantic identity shifts
- betrayal or deception memory
- health disruption
- academic delay or failure
- money shame and resource pressure
- care responsibility
- relocation and belonging split

Each deposit carries:

- age stage multiplier
- repair signal
- variable effects on the present life state
- branch effects on inertia, intervention, and rupture lines
- a human-readable life effect for the report

This layer is how the model moves from "current variable explanation" toward "a life gradually shaped by accumulated events."

## 5. Inner Agents

Inner agents are not static labels.

Each agent has:

- desire
- fear
- strategy
- memory
- trigger conditions
- influence score
- stage-specific behavior

## 6. Personal World State

To simulate fate rather than only personality, the engine also models external reality:

- social roles
- personal assets
- city and industry environment
- family system
- relationship network
- health and energy
- time horizon

Derived scores:

- role load
- financial runway
- external pressure
- support capacity
- energy risk
- life mobility

## 7. Conflict Rules

Conflict rules describe how agents interact.

Example:

```text
The Controller + The Observer
→ clear analysis, delayed entry
→ high risk of staying before the threshold of action
```

## 8. Life Environment

The environment represents external pressures:

- career market
- family expectations
- intimacy demands
- economic constraints
- cultural scripts
- social support

## 9. Parallel Simulation

The engine should generate multiple branches:

- inertia branch: current pattern continues
- intervention branch: one key behavior changes
- rupture branch: suppressed conflict breaks through

## 10. Report Synthesis

The report should include:

- dominant agents
- core conflict
- life-stage risks
- parallel trajectories
- inflection points
- reflection prompts
